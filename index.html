01

Overview

Three.js is a JavaScript library built on top of WebGL that makes creating 3D graphics in the browser approachable. It abstracts the low-level GPU API into a clean scene graph with objects, cameras, lights, materials and a render loop.

02

Geometry

Geometry defines the shape of a 3D object — the vertices, edges and faces that make up its surface. Three.js provides 21 built-in geometry types covering everything from simple boxes to complex parametric surfaces. All geometry is stored as a BufferGeometry — a flat typed array of vertex attributes (position, normal, UV, color) sent directly to the GPU.

↗ BufferGeometry ↗ BoxGeometry ↗ Examples

All 21 Built-in Geometry Types

BoxGeometry, SphereGeometry, CylinderGeometry, ConeGeometry, TorusGeometry, TorusKnotGeometry, PlaneGeometry, CircleGeometry, RingGeometry, DodecahedronGeometry, IcosahedronGeometry, OctahedronGeometry, TetrahedronGeometry, CapsuleGeometry, LatheGeometry, ExtrudeGeometry, ShapeGeometry, TubeGeometry, EdgesGeometry, WireframeGeometry, PolyhedronGeometry.

03

Materials

Materials control how geometry looks when rendered — its color, shininess, transparency, how it reacts to light. Three.js has materials for every use case from the cheapest flat-shaded MeshBasicMaterial to the fully physics-based MeshPhysicalMaterial with clearcoat, iridescence, transmission and subsurface scattering.

↗ Material base ↗ MeshStandardMaterial ↗ MeshPhysicalMaterial

All Material Types

MeshBasicMaterial (no lighting), MeshLambertMaterial (diffuse), MeshPhongMaterial (specular), MeshStandardMaterial (PBR), MeshPhysicalMaterial (advanced PBR), MeshToonMaterial (cel-shading), MeshNormalMaterial (debug), MeshDepthMaterial, MeshMatcapMaterial, LineBasicMaterial, LineDashedMaterial, PointsMaterial, SpriteMaterial, ShadowMaterial, ShaderMaterial, RawShaderMaterial.

04

Lighting

Lighting is what makes a 3D scene feel real. Three.js offers 7 light types from the flat uniform AmbientLight to the precise projecting SpotLight. Only DirectionalLight, SpotLight and PointLight cast shadows — each using a shadow map technique where the scene is rendered from the light's perspective to determine what's in shadow.

↗ Light base ↗ DirectionalLight ↗ Shadows

7 Light Types + Shadow Maps

AmbientLight (uniform fill, no shadows), DirectionalLight (sun — parallel rays, shadows), PointLight (bulb — radiates in all directions, shadows), SpotLight (flashlight — cone shape, shadows), HemisphereLight (sky + ground gradient), RectAreaLight (soft box light, no shadows), LightProbe (spherical harmonics from environment).

05

Scene Objects

Everything in a Three.js scene is an Object3D — the base class that provides position, rotation, scale, and parent-child hierarchy. From basic Mesh to GPU-instanced InstancedMesh, point clouds, billboards and level-of-detail systems, each object type serves a specific purpose.

↗ Object3D ↗ Mesh ↗ InstancedMesh

All Object Types

Mesh (geometry + material), InstancedMesh (1 draw call, N instances), BatchedMesh (r159+, mixed geometry batching), SkinnedMesh (skeletal animation), Points (particle clouds), Line / LineSegments / LineLoop, Sprite (always-facing billboard), Group (parent transform container), LOD (level of detail by distance), Scene (root container).

07

Cameras

The camera defines what part of the 3D scene is projected onto the 2D screen. PerspectiveCamera is the standard — it mimics human vision with a field-of-view frustum. OrthographicCamera is used for isometric games and UI because objects don't get smaller with distance. The others are specialised: CubeCamera captures reflections, ArrayCamera renders split-screen, StereoCamera renders VR.

↗ PerspectiveCamera ↗ OrthographicCamera ↗ CubeCamera

PerspectiveCamera — Field of View

The most common camera. Takes fov, aspect, near, far. Low FOV (30°) = telephoto zoom, objects look flat and close. High FOV (120°) = ultra-wide, strong perspective distortion, see more of the scene. Standard FPS games use 75–90°.

OrthographicCamera — No Perspective

Objects the same size regardless of distance. Perfect for isometric games, 2D UIs, CAD-style views and shadow maps. Takes left, right, top, bottom, near, far to define a box-shaped frustum.

CubeCamera — 360° Environment Capture

Renders the scene 6 times (±X, ±Y, ±Z) to create a cubemap. Used for real-time reflections and refractions — position it inside an object to capture what it should reflect.

ArrayCamera — Multiple Viewports

An array of cameras each rendering to a specific viewport region. Used for split-screen multiplayer, security camera UIs, or rendering the same scene from multiple angles simultaneously.

StereoCamera — VR / 3D Anaglyph

Two cameras with a configurable eye separation offset, rendering left and right eye views. Used with VR headsets or classic red-cyan anaglyph 3D glasses.

08

Textures

Textures are images mapped onto geometry surfaces to add detail without adding geometry. Three.js supports many texture types: basic colour maps, normal maps for fake surface detail, PBR texture sets, video textures, HDR environments and compressed formats like KTX2 and Basis for smaller file sizes.

↗ Texture ↗ TextureLoader ↗ CubeTexture

Texture Basics & UV Mapping

How 2D images are mapped onto 3D surfaces via UV coordinates. Each vertex has a UV coordinate (0–1 range) that tells the GPU which part of the texture to sample there.

Normal Maps — Fake Surface Detail

RGB images that encode surface normals, tricking the lighting model into thinking there is real surface detail. Dramatically increases visual complexity without adding geometry.

PBR Texture Set

A full physically-based rendering texture set: Albedo (base colour), Normal (surface detail), Roughness (how blurry reflections are), Metalness (is it metal?), Ambient Occlusion (contact shadows). Used by MeshStandardMaterial and MeshPhysicalMaterial.

CubeTexture — Skybox

6 images (±X, ±Y, ±Z faces) assembled into a cubemap. Set as scene.background for a skybox or scene.environment for IBL (image-based lighting).

VideoTexture — Live Video on Surfaces

Maps a playing HTML <video> element onto a 3D surface in real time. Used for in-game screens, TVs, portals and video walls.

DataTexture / DataArrayTexture

Create textures directly from raw typed array data in JavaScript. Used for scientific visualisations, heatmaps, heightmaps and GPU-computed data.

HDR / EXR Environment Maps

High dynamic range images that store light values above 1.0. Used as IBL environments — they realistically light all objects in the scene based on the real lighting captured in the HDRI.

Compressed Textures — KTX2 / Basis

GPU-native compressed formats that stay compressed in VRAM, using 4–8× less memory than PNG. KTX2 with Basis supercompression is the modern standard — transcodes to the best format per device (DXT, ETC, ASTC).

09

Renderer Features

The renderer is the engine that turns your scene into pixels. Three.js has both WebGLRenderer (the proven standard) and WebGPURenderer (the modern future). Both expose a common abstraction layer so most code works with either. Key features include shadow maps, environment mapping, tone mapping and anti-aliasing.

↗ WebGLRenderer ↗ WebGPURenderer ↗ Constants

Shadow Map Types

BasicShadowMap (fast, aliased), PCFShadowMap (smoothed via percentage-closer filtering), PCFSoftShadowMap (softer), VSMShadowMap (variance — soft, no peter-panning). Set via renderer.shadowMap.type.

Environment Mapping

Objects reflecting the surrounding environment via cubemap or PMREM. Set scene.environment to automatically light all PBR materials with IBL (image-based lighting).

Tone Mapping

Converts HDR light values to displayable LDR range. NoToneMapping (clamps), LinearToneMapping, ReinhardToneMapping (photographic), CineonToneMapping, ACESFilmicToneMapping (filmic S-curve), AgXToneMapping (r158+, physically accurate).

Anti-Aliasing

Smooths jagged edges. Built-in: MSAA (pass antialias:true to renderer). Post-process: FXAA (fast, slight blur), SMAA (higher quality). TAA (temporal) available via post-processing for best quality.

Render Targets — Off-screen Rendering

Render the scene to a texture instead of the screen. The texture can then be used as input to another pass, applied to a 3D surface (in-game screens), or processed by post-processing effects.

WebGL vs WebGPU

WebGL uses the OpenGL ES 2.0/3.0 API (older, widely supported). WebGPU is the modern GPU API with compute shaders, better multi-threading and explicit resource management. Three.js supports both via a common renderer abstraction.

Compute Shaders (WebGPU)

GPU programs that run outside the graphics pipeline for general computation — simulating millions of particles, fluid dynamics, physics. Available via WebGPURenderer with TSL's compute() node.

Render Pipeline Overview

Scene graph → frustum culling → render list sorted by material → draw calls → vertex shader → rasterisation → fragment shader → framebuffer → screen. Understanding this pipeline is key to performance.

10

Post-Processing Effects

Post-processing applies screen-space effects after the scene is rendered. Three.js uses EffectComposer with a chain of passes — the output of one feeds into the next. The library ships with over 20 built-in effects, and you can write custom fullscreen shader passes for complete control.

↗ EffectComposer ↗ Examples

EffectComposer — The Pipeline

Orchestrates a chain of passes. RenderPass renders the scene. Subsequent passes read from and write to render targets. OutputPass writes the final result to screen. Add passes in order: scene → effects → output.

Bloom / UnrealBloomPass

Bright areas of the scene bleed light into surrounding pixels, simulating the glow of bright lights on a camera lens. Adjust threshold, strength and radius. Essential for neon, HDR and sci-fi aesthetics.

SSAO — Screen Space Ambient Occlusion

Darkens crevices, corners and contact areas where ambient light can't reach. Gives scenes grounding and depth without expensive ray-traced ambient occlusion. SAOPass is the Three.js implementation.

Depth of Field (Bokeh)

Blurs objects outside the focal plane, mimicking a physical camera lens. Objects close to the focus distance are sharp; others are blurred with bokeh discs. BokehPass or postprocessing library's DoF effect.

Film Grain

Adds animated noise to the final frame, simulating photographic film grain. Softens the overly-clean look of CGI and adds cinematic character. FilmPass in Three.js includes grain + scanlines.

Glitch Effect

Digital corruption effect — random horizontal band shifts, RGB channel separation, noise overlays. GlitchPass generates random glitch bursts. Great for cyberpunk and error/hack aesthetics.

Outline — Selection Highlight

OutlinePass renders selected objects with a coloured outline. Works by rendering object silhouettes at a slight scale offset. Essential for game selection UIs and highlighting interactive objects.

Color Grading / LUT

Applies a 3D Look-Up Table to remap colours in the final frame, matching a specific cinematic colour grade. LUTPass in Three.js — load a .cube or .png LUT file to transform the entire scene's mood instantly.

Motion Blur

Blurs fast-moving objects along their trajectory, as a real camera shutter would capture. Can be per-object (velocity-based) or camera motion blur. Gives movement weight and speed.

God Rays — Volumetric Light

Shafts of light visible when a bright source (sun, lamp) is partially occluded. GodraysPass renders the scene from the light's perspective then scatters the result radially toward the screen.

Chromatic Aberration

Offsets the red, green and blue colour channels by slightly different amounts, creating colour fringing at high-contrast edges — mimicking cheap or damaged camera lenses. Lo-fi and retro aesthetic.

Pixel / Retro Effect

Downsamples the scene to a low resolution then upscales it with nearest-neighbour filtering, creating a pixel art aesthetic. Combine with a limited colour palette for a full retro game look.

11

Controls

Controls add interactivity — they translate mouse, keyboard and touch input into camera or object movement. Three.js ships 8+ control types via the addons (jsm/controls). The right control depends entirely on the use case: exploration, FPS, flight, map viewing or object manipulation.

↗ OrbitControls ↗ TransformControls ↗ PointerLockControls

OrbitControls

Camera orbits around a target point. Left-drag rotates, right-drag pans, scroll zooms. The default for 3D model viewers and product configurators. Supports damping for smooth movement.

FirstPersonControls

Mouse look + WASD movement. The camera moves through the scene from the player's eye perspective. Suitable for walk-throughs and exploration. Note: PointerLockControls is preferred for games.

FlyControls

6 degrees of freedom — move forward/back, strafe, yaw, pitch, roll. No ground plane constraint. Ideal for space exploration, drone views and flight simulators.

MapControls

Variant of OrbitControls optimised for top-down map panning. Right-click rotates, left-click pans, scroll zooms. The camera always points roughly downward. Ideal for strategy games and GIS apps.

TransformControls

Attaches a gizmo to a 3D object allowing direct manipulation — translate (arrows), rotate (rings), scale (handles). Essential for any in-browser 3D editor. Press T/R/S to switch modes.

PointerLockControls

Locks the mouse cursor (hides it, raw mouse delta only) for a true FPS game feel. The Pointer Lock API must be triggered by a user gesture (click). Mouse movement directly rotates the camera view.

12

Node System / TSL (Three Shading Language)

TSL is Three.js's visual shader authoring system — write shaders in JavaScript using composable node functions that compile to GLSL for WebGL or WGSL for WebGPU. No raw shader strings, no GLSL knowledge required. Nodes are composable functions: color(), texture(), add(), normalMap(), noise(). Chain them to build any shader effect.

↗ TSL Wiki ↗ NodeMaterial ↗ TSL Editor

What is TSL?

Connect JavaScript node functions → get a compiled GLSL/WGSL shader. No shader strings, no uniform management, no compile errors from typos. Just composable JS functions that describe the shader graph.

Noise Functions

mx_noise_float() for Perlin-style noise, mx_worley_noise() for cellular patterns, checker(), mx_fractal_noise() for FBM (fractal Brownian motion). All run on GPU, great for terrain, clouds, water, fire.

Math Operations

add(), sub(), mul(), div(), mix(), clamp(), smoothstep(), pow(), abs(), sin(), cos(), floor(), fract(), length(), dot(), cross(), normalize() — all available as TSL nodes that compile to GPU shader code.

Fresnel / Rim Lighting

The Fresnel effect makes surfaces more reflective at glancing angles — water, glass, car paint. In TSL: fresnel() node gives a 0–1 value based on view angle. Mix it into colour or emission for rim glow.

Displacement / Vertex Animation

Move vertices in the vertex shader based on noise or texture values. Assign a node to material.positionNode to animate geometry on GPU — terrain heightmaps, ocean waves, cloth, breathing characters.

Triplanar Mapping

Projects texture from X, Y and Z axes simultaneously and blends them — allows texturing complex or irregular geometry without UV unwrapping. triplanarTexture() node in TSL.

PBR Node Breakdown

How texture nodes feed into MeshPhysicalNodeMaterial properties: texture() → colorNode, normalMap() → normalNode, roughnessMap → roughnessNode, metalnessMap → metalnessNode. Full PBR pipeline in TSL.

GPGPU — General Purpose GPU

Use compute() nodes in WebGPU mode to run arbitrary GPU computation — simulate millions of particles, fluid, cloth or crowds fully on the GPU. Read results back into render geometry each frame.

Procedural Shaders

Generate patterns entirely in the shader with no textures — checkerboards, gradients, grids, mandalas, fractal noise. All computed per-pixel on the GPU from mathematical functions.

Custom Shader — Full Control

Assign any TSL expression to colorNode, emissiveNode, positionNode, normalNode etc. Full control over every aspect of the material. Or use ShaderMaterial/RawShaderMaterial with raw GLSL for maximum control.

13

Animation

Three.js has a complete animation system based on keyframes. AnimationClip stores a sequence of keyframe tracks. AnimationMixer plays clips on a specific object, handling blending between multiple clips. Skeletal animation deforms mesh vertices via a bone hierarchy. Morph targets blend between pre-baked vertex positions.

↗ AnimationMixer ↗ AnimationClip ↗ SkinnedMesh

Keyframes & Tracks

Keyframes are values at specific times. A KeyframeTrack stores the name of the property (.position.x), an array of times, and an array of values. The interpolant (linear, step, cubic) fills in the gaps between keyframes.

Walk Cycle

A looping animation of roughly 8 key poses that creates the illusion of walking. The poses are keyframed, AnimationMixer plays the clip in a loop, and the transition from last frame to first frame is seamless.

Blend & Crossfade

AnimationMixer can play multiple actions simultaneously and crossfade between them. action.crossFadeTo(other, duration) smoothly transitions weight from one clip to another — walk → run, idle → attack.

Morph Targets (Blend Shapes)

Pre-baked vertex positions stored on the geometry. mesh.morphTargetInfluences[i] = 0..1 blends between base and target shape. Used for facial expressions, speech lip-sync and secondary deformations.

Skeletal Animation

A Skeleton of Bones deforms the SkinnedMesh via skinning weights — each vertex is influenced by 1–4 bones with blended weights. Bone transforms are keyframed via the animation system.

Additive Animation

An additive clip adds its deltas on top of a base pose rather than replacing it. Breathing, look-around, flinch reactions can all be additive layers on top of the base locomotion — action.blendMode = THREE.AdditiveAnimationBlendMode.

Animation Curves — Interpolation

How values change between keyframes: InterpolateLinear (straight line), InterpolateSmooth (cubic Catmull-Rom), InterpolateDiscrete (instant step). Ease-in, ease-out and custom cubic bezier easing via AnimationUtils.

AnimationMixer Concept

One mixer per animated object. Call mixer.clipAction(clip) to get an AnimationAction. Set action.play(), action.weight, action.timeScale. Call mixer.update(delta) every frame. Multiple actions blend automatically.

14

Audio

Three.js integrates with the Web Audio API to provide 3D spatial audio. AudioListener is attached to the camera. PositionalAudio sources are attached to 3D objects — they automatically adjust volume and panning based on distance and angle from the listener. AudioAnalyser exposes frequency data for reactive visualisations.

↗ Audio ↗ PositionalAudio ↗ AudioAnalyser

Spatial Audio — 3D Positioning

AudioListener on camera, PositionalAudio on each sound source. The Web Audio PannerNode automatically pans and attenuates based on the relative position. Walk closer — louder. Walk around — panning shifts.

Audio Falloff — Distance Curves

Controls how volume drops with distance. rolloffFactor and distanceModel (linear, inverse, exponential) set the shape of the curve. refDistance is where volume starts dropping; maxDistance is where it reaches 0.

AudioAnalyser — Frequency Data

Wraps Web Audio AnalyserNode to extract FFT frequency data as a Uint8Array. getFrequencyData() each frame — use the values to animate geometry, particle sizes, emissive intensity or anything else that should react to music.

Audio Cones — Directional Sound

PositionalAudio supports cone-based directional audio. coneInnerAngle (full volume zone), coneOuterAngle (volume transitions), coneOuterGain (volume at and beyond outer cone). Perfect for directional speakers, NPCs, vehicles.

15

Special Objects & Effects

Three.js addons include high-level objects for common visual needs — ocean water, procedural sky, mirror reflections, lens flares, HTML-in-3D, and powerful rendering utilities. These are in jsm/objects/ and work with both WebGL and WebGPU renderers.

↗ Water Example ↗ Sky Example ↗ Reflector

Water

Ocean/lake water with animated normal maps, reflections and refractions. Water2 supports two normal map textures for more complex wave patterns. Can adjust waterColor, sunDirection, distortionScale, size.

Sky

Procedural Preetham sky model. turbidity, rayleigh, mieCoefficient, mieDirectionalG and sunPosition parameters control everything from dawn pink to midday blue to sunset orange. Works without any textures.

Reflector — Mirror Effect

Renders the scene from a reflected camera perspective onto a plane, creating a real-time mirror. Used for floors, mirrors, water surfaces. ReflectorForSSRPass adds screen-space reflections.

Lensflare

Adds lens flare artefacts around bright lights — the hexagonal aperture shapes seen when a bright light hits a camera lens. Attach Lensflare to a light source with textured flare elements at different offsets.

CSS3DObject — HTML in 3D Space

CSS3DRenderer overlays HTML elements transformed to match 3D scene positions. Embed live web pages, interactive UI, video iframes and DOM elements directly into your 3D scene.

Instancing — 100k Objects, 1 Draw Call

InstancedMesh renders N copies of the same geometry+material in a single GPU draw call. Each instance has its own matrix (position, rotation, scale) and optional colour. Rendering forests, crowds, particles, asteroids.

LOD — Level of Detail

Automatically swaps between high/medium/low poly versions of an object based on camera distance. LOD.addLevel(mesh, distance) — the closest level uses the most detailed geometry; far objects use simplified versions.

Sprite Particles

Points geometry + PointsMaterial renders each vertex as a billboard sprite. size, sizeAttenuation (perspective scaling), map (texture), vertexColors allow rich particle systems — fire, smoke, rain, sparkles, explosions.

Raycaster — Mouse Picking

Cast a ray from camera through the mouse pointer into the scene. raycaster.setFromCamera(pointer, camera) then intersectObjects() returns hit objects sorted by distance. The basis for all 3D mouse interaction: hover, click, drag.

Clipping Planes

Discard fragments on one side of a mathematical plane. renderer.clippingPlanes clips globally. material.clippingPlanes clips per material. Combine planes to cut complex cross-sections through objects — great for medical/architecture visualisation.

16

Loaders

Loaders import external assets — 3D models, textures, environments. Three.js ships with a base Loader class and dozens of format-specific loaders in the addons. GLTFLoader is the recommended format for 3D models — it's the "JPEG of 3D", compact, supports PBR materials, animation, morphs and extensions.

↗ GLTFLoader ↗ Loader base ↗ DRACOLoader

GLTFLoader — The Standard Format

.gltf (JSON + binary + images) or .glb (single binary blob). Supports PBR materials, skeletal animation, morph targets, multiple meshes, lights, cameras, custom extensions. Use Draco or meshopt compression for smaller files.

FBXLoader — Autodesk Format

Autodesk FBX format common in game pipelines (Maya, 3ds Max, MotionBuilder). Supports skeletal animation and morph targets. Large files — prefer exporting to GLTF from DCC tools when possible.

OBJLoader — Universal Legacy Format

The most widely supported 3D format. Human-readable text, paired with .mtl for materials. No animation, no PBR, large files. Good for static geometry interchange with any 3D application.

Draco Compression

Google's open-source mesh compression — reduces geometry file size by 90%+ with minimal visual loss. Embed in GLTF via KHR_draco_mesh_compression. Requires DRACOLoader + decoder WASM loaded separately.

RGBELoader / EXRLoader — HDR Environments

Load .hdr (Radiance RGBE format) or .exr (OpenEXR) HDRI environment maps. Pass to PMREMGenerator.fromEquirectangular() to prefilter for physically-based IBL lighting and reflections.

SVGLoader — Vector Graphics in 3D

Parses SVG paths into Three.js ShapePaths. Extrude shapes with ExtrudeGeometry for 3D logos and text, or render flat as ShapeGeometry. Preserves curves and fill colours from the SVG.

17

Physics

Three.js doesn't include a physics engine — but it pairs seamlessly with popular libraries. Rapier (WASM, very fast), Cannon.js / cannon-es (pure JS), Ammo.js (Bullet physics port), and Jolt are the most common choices. The pattern is always the same: physics world runs the simulation, Three.js renders the result.

↗ Rapier ↗ cannon-es ↗ Ammo.js

Rigid Body Physics

Objects with mass that fall, bounce, slide and collide. Each physics body has a shape (box, sphere, convex hull), mass, friction, restitution. Every frame: physics.step() → copy body.position to mesh.position.

Cloth Simulation

A grid of connected particles with spring constraints. Pin some particles (corners), apply gravity and wind forces, resolve constraints iteratively each frame. Drapes naturally over objects. Ammo.js soft body or custom verlet integration.

Vehicle Physics

Ray-cast vehicle simulation: a chassis rigid body with ray-cast wheels. Rays cast downward detect the ground surface; suspension compresses accordingly. Engine force applied to driven wheels, steering to front wheels.

Constraints & Ragdoll

Constraints connect two rigid bodies with limits on their relative movement — hinge (door), slider (piston), lock (weld), cone-twist (shoulder joint). Chain them to build ragdolls, wrecking balls, robotic arms.

18

Helpers & Debug Tools

Helper objects are added to the scene during development to visualise invisible things — camera frustums, light directions, bone positions, bounding boxes. They are Object3D instances you add and remove like any other object. Always remove them before shipping.

↗ GridHelper ↗ CameraHelper ↗ SkeletonHelper

GridHelper + AxesHelper

GridHelper draws a flat grid at Y=0 — orient yourself in 3D space. AxesHelper draws the XYZ axes as red/green/blue lines from origin. Combine them to instantly visualise the coordinate system.

CameraHelper

Renders the wireframe frustum of a camera as lines in the scene. Attach to any camera to visualise its field of view, near plane and far plane. Essential when debugging shadow cameras and splitscreen setups.

Light Helpers

DirectionalLightHelper (arrow + target line), PointLightHelper (wireframe sphere), SpotLightHelper (cone wireframe), HemisphereLightHelper (two hemispheres), RectAreaLightHelper (outlined rectangle).

SkeletonHelper

Renders the bone hierarchy of a SkinnedMesh as coloured lines showing each bone's position and orientation. Essential for debugging animation rigs, verifying bone names and checking skeleton hierarchy.

19

Curves, Shapes & Extras

Three.js has a rich 2D/3D curve and shape system. Curves define smooth paths — use them for camera paths, tube geometry, spline animations. Shapes define 2D outlines that can be extruded into 3D geometry. PMREMGenerator prefilters environment maps for physically-based rendering.

↗ CatmullRomCurve3 ↗ Shape ↗ PMREMGenerator

Bezier Curves

QuadraticBezierCurve3 (1 control point) and CubicBezierCurve3 (2 control points). Define smooth curves through 3D space. .getPoints(n) samples N points along the curve — use with TubeGeometry to render them.

CatmullRom Spline — Camera Paths

Smooth curve that passes through all control points (unlike bezier which may not). Perfect for cinematic camera paths — sample getPointAt(t) each frame where t goes 0→1 over the animation duration.

Shape + ExtrudeGeometry

Define a 2D outline with Shape (moveTo, lineTo, bezierCurveTo, arc). Pass to ShapeGeometry for flat 2D, or ExtrudeGeometry to add depth and bevel. Great for custom logos, text, architectural elements.

PMREMGenerator

Pre-filters a raw environment map (HDRI or cubemap) into a Prefiltered Mipmapped Radiance Environment Map. The different mip levels correspond to different roughness values — rough surfaces sample blurry mips, mirrors sample sharp ones. Required for correct PBR rendering.

14

Math Library

Three.js includes a complete 3D math library. These classes are used everywhere — setting positions, rotating objects, computing transforms, raycasting. Understanding Vector3, Quaternion and Matrix4 is fundamental to working with Three.js at any level beyond the basics.

↗ Vector3 ↗ Quaternion ↗ Matrix4 ↗ Euler ↗ Box3

Complete Math Library

Vector2/3/4, Matrix3/4, Quaternion (rotation without gimbal lock), Euler (human-readable rotation angles), Color (RGB + HSL), Ray (origin + direction for raycasting), Plane (infinite mathematical plane), Box2/3 (axis-aligned bounding boxes), Sphere, Frustum, Triangle, Line3, MathUtils (lerp, clamp, degToRad, seededRandom, etc.)
